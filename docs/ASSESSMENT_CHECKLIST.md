# SEAPEDIA Assessment Checklist

## Level 1

- Public landing page: `/`
- Public product listing: `/#/products`
- Product search, filter, and sort: `GET /api/products?q=&store_id=&min_price=&max_price=&in_stock=&sort=`
- Product detail: `/#/products/{id}`
- Public store detail: `/#/stores/{id}` and `/api/stores/{store_id}`
- Login and register: `/#/login`, `/#/register`
- Password hashing: PBKDF2-HMAC-SHA256
- JWT auth with active role
- Multi-role account: `multi / Multi123`
- Role selection: `/#/choose-role`
- Frontend Vue.js app: `frontend/src/App.vue`
- Conventional Express structure: `backend/src/app.js`, `routes/`, `controllers/`, and `middleware/`
- Reusable UI components: cards, buttons, inputs, navbar, panel, data table, timeline, dashboard shell
- Role-aware notification center: `/api/notifications`

## Level 2

- Seller store management: `/api/seller/store`
- Unique store name validation
- Product CRUD: `/api/seller/products`
- Product ownership checks
- Public catalog connected to backend data

## Level 3

- Buyer wallet: `/api/buyer/wallet`
- Wallet top-up: `/api/buyer/wallet/topup`
- Delivery address: `/api/buyer/addresses`
- Cart: `/api/buyer/cart`
- Single-store checkout rule enforced server-side
- Checkout summary and order creation
- PPN 12%, delivery fee, subtotal, final total
- Stock reduction prevents negative stock
- Buyer order history and Seller incoming order list
- Explicit Buyer order detail: `/api/buyer/orders/{order_id}` and `/#/orders/{order_id}`

## Level 4

- Voucher and Promo resources
- Admin discount creation endpoints
- Admin discount list and detail endpoints
- Discount validation during checkout
- Seller process order from `Sedang Dikemas` to `Menunggu Pengirim`
- Buyer and Seller reports
- Order timeline with timestamps

## Level 5

- Delivery job resource
- Driver available jobs
- Driver job detail endpoint and UI detail block
- Driver take job
- Driver complete job
- One Driver per order
- Driver earnings and job history

## Level 6

- Admin monitoring dashboard
- Users, stores, products, orders, vouchers, promos, delivery jobs, overdue orders
- Admin product monitoring endpoint: `/api/admin/products`
- Admin delivery-job monitoring endpoint: `/api/admin/delivery-jobs`
- Admin analytics endpoint: `/api/admin/analytics`
- Admin audit trail endpoint: `/api/admin/audit-logs`
- Voucher and Promo management UI
- Voucher detail: `/api/admin/vouchers/{voucher_id}`
- Promo detail: `/api/admin/promos/{promo_id}`
- SLA rules by delivery method
- SLA day progression endpoint
- Auto return/refund to `Dikembalikan`
- Wallet refund, stock restoration, status history, double refund prevention

## Level 7

- SQL Injection prevention by avoiding raw SQL and validating JSON-store mutations
- XSS prevention by escaping public comments and dynamic frontend values
- Input validation through Node.js request handlers
- Active role enforced server-side
- Ownership authorization checks
- API documentation at `/docs` and OpenAPI JSON at `/openapi.json`
- Seed accounts available
- README includes testing guide and security notes
- Automated end-to-end smoke test: `npm test`
- Docker support for backend Express and frontend Vue

## Value-Added Features

- Catalog search/filter/sort for a more realistic marketplace browsing flow.
- Store detail page with products, completed order count, and income summary.
- Notification center generated by checkout, seller processing, driver delivery, completion, and overdue refund events.
- Admin analytics for GMV, completed GMV, PPN, discounts, seller income, driver earnings, active orders, low stock, and store performance.
- Admin audit trail records actor, active role, action, entity, details, and timestamp for critical system mutations.
